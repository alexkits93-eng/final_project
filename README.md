# Вторая часть практического задания: Автоматизация теста к API

## Описание:
Автоматизация сценария:
1. Клиент создает заказ.
2. Проверяется, что по треку заказа можно получить данные о заказе.

## Структура проекта
- `configuration.py` - содержит URL и пути к API эндпоинтам
- `data.py` - тестовые данные и заголовки запросов
- `stand_requests.py` - функции для отправки запросов к API
- `scooter_autotest.py` - автотест сценария "создание заказа → получение по треку"
- `screen` - скриншот результата выполнения теста

## Шаги автотеста:
- Выполнить запрос на создание заказа.
- Сохранить номер трека заказа.
- Выполнить запрос на получение заказа по треку заказа.
- Проверить, что код ответа равен 200.

## Запуск теста
### Если установлены библиотеки requests, pytest
pytest scooter_autotest.py -v

### Если не установлены библиотеки requests, pytest
pip install requests pytest

pytest scooter_autotest.py -v
